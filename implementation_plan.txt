Supporto Registri Modbus 32-bit Float con Virgola
Questo piano descrive le modifiche per abilitare la gestione di valori floating point (IEEE 754 a 32 bit, precisione singola) per i registri Modbus Holding Register (hr), che hanno passo/dimensione 2. Attualmente questi registri vengono trattati come interi a 32 bit. La modifica coprirà l'intera catena di comunicazione: client Modbus, loop di lettura, loop di scrittura, backend API del frontend e interfaccia utente.

User Review Required
IMPORTANT

Le modifiche verranno applicate sia ai file sorgente in modbus-gateway/source/ sia ai file di runtime in modbus-gateway/. Questo sostituirà i file criptati tramite Pyarmor presenti in modbus-gateway/ con codice sorgente in chiaro in modo da consentire il funzionamento e i test. Il cliente potrà eseguire nuovamente l'offuscamento con Pyarmor al momento del rilascio in produzione.

NOTE

La precisione decimale visualizzata e accettata sarà limitata ad un massimo di 5 decimali, come richiesto.

Proposed Changes
Componente: Modbus Gateway (Backend Python)
[MODIFY] 
modbus_client.py
 (e runtime 
modbus_client.py
)
Modificare read_holding_register_32bit per interpretare le due word a 16-bit come float a 32-bit big-endian (>f) tramite il modulo struct, anziché come intero.
Modificare write_holding_register_32bit per accettare un valore float, convertirlo in rappresentazione binaria IEEE 754 a 32-bit big-endian, ed estrarne le due word a 16-bit per la scrittura Modbus.
[MODIFY] 
modbus_reader.py
 (e runtime 
modbus_reader.py
)
Modificare _read_hr_with_split per convertire le word lette in valori float usando struct.unpack('>f', ...).
Modificare _normalize_value in modo da non eseguire il cast a int per i registri di tipo hr (mantenendo il valore float originale con i decimali).
[MODIFY] 
modbus_writer.py
 (e runtime 
modbus_writer.py
)
Modificare _validate_value per validare l'input come float (ammettendo valori negativi e controllando il limite di 5 cifre decimali).
Modificare cycle per rimuovere il cast a int prima del salvataggio nel database e della scrittura sul client Modbus.
Componente: Modbus Frontend (Flask App + UI)
[MODIFY] 
app.py
Modificare validate_register_value per convalidare i registri hr come float anziché come interi, con controllo sui range standard di un float a 32 bit e sul limite massimo di 5 decimali.
[MODIFY] 
index.html
Modificare la funzione fmt(v) in Javascript per visualizzare al massimo 5 cifre decimali, rimuovendo gli zeri superflui finali per una resa visiva ottimale.
Modificare il tag <input type="number"> rimpiazzando min="0" max="4294967295" con step="any" per sbloccare l'inserimento dei decimali nel browser.
Modificare handleRwInput per accettare valori con virgola/decimali e validare il limite di massimo 5 decimali.
Aggiungere il badge grafico FLOAT di colore verde/azzurro accanto a HR nella schermata dei registri Read-Only e Writable per segnalare all'utente che il registro supporta valori float con virgola.
Verification Plan
Test Manuali / Simulazione
Creeremo uno script di mock/test Modbus Server locale (in esecuzione sulla porta di test 8001) per simulare la lettura e la scrittura di registri float a 32 bit.
Avvieremo il Gateway (modbus_reader.py e modbus_writer.py) e il Frontend (app.py) puntando al server di test locale.
Verificheremo da interfaccia web:
Che i registri HR mostrino l'indicazione FLOAT.
Che sia possibile inserire valori come 123.45 o 10.00001 (5 decimali).
Che valori con più di 5 decimali (es. 12.123456) vengano respinti con indicazione di errore.
Che i valori vengano trasmessi correttamente al server di test Modbus e memorizzati nel database.
